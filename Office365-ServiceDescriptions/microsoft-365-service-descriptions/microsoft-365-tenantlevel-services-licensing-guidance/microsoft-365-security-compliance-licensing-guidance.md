# <a name="microsoft-365-guidance-for-security-amp-compliance"></a>Microsoft 365 по обеспечению соответствия требованиям безопасности &amp;

Для целей этой статьи служба на уровне клиента — это веб-служба, которая активируется частично или полностью для всех пользователей в клиенте (отдельная лицензия и (или) как часть плана Microsoft 365 или Office 365). Соответствующие лицензии на подписку необходимы для использования клиентом веб-службы. Чтобы просмотреть варианты лицензирования пользователей, чтобы воспользоваться преимуществами Microsoft 365 соответствия требованиям, скачайте таблицу Microsoft 365  [сравнения](https://go.microsoft.com/fwlink/?linkid=2139145).

Некоторые службы клиента в настоящее время не могут ограничивать преимущества определенными пользователями. Необходимо предпринять усилия, чтобы ограничить преимущества службы лицензированными пользователями. Дополнительные сведения об условиях использования продуктов Майкрософт и Professional Services, полученных с помощью программ лицензирования Майкрософт, см. в разделе "Условия продукта ["](https://www.microsoft.com/Licensing/product-licensing/products).

## <a name="advanced-audit"></a>Расширенный аудит

Расширенный аудит в Microsoft 365 обеспечивает хранение журналов аудита для действий пользователей и администраторов на один год и позволяет создавать настраиваемые политики хранения журналов аудита для управления хранением журналов аудита для других служб Microsoft 365. Он также предоставляет доступ к важным событиям для исследования и доступу с высокой пропускной способностью к API Office 365 действий управления. Дополнительные сведения см. в [разделе "Расширенный аудит" Microsoft 365](/microsoft-365/compliance/advanced-audit).

Вы также можете включить срок хранения в 10 лет с помощью номера SKU надстройки.

### <a name="which-users-benefit-from-the-service"></a>Какие пользователи получают преимущества от этой службы?

Пользователи с лицензиями Office 365 E5/A5/G5, Microsoft 365 E5/A5/G5, Microsoft 365 E5/A5/G5/F5, Microsoft 365 F5 Security & Compliance и Microsoft 365 E5/A5/G5 eDiscovery и Audit могут воспользоваться расширенным аудитом.

Лицензированные пользователи с расширенным аудитом и надстройкой 10-year Audit Log Retention могут воспользоваться преимуществами 10-летного хранения журнала аудита.

### <a name="how-do-users-benefit-from-the-service"></a>Как пользователи получают преимущества от этой службы?

Пользователи получают преимущества расширенного аудита, так как записи аудита, связанные с действиями Microsoft 365 служб, могут храниться до одного года. Кроме того, регистрируются события аудита с высокой ценностью, например при доступе или считывания элементов в почтовом ящике пользователя. Дополнительные сведения см. в [разделе "Расширенный аудит" Microsoft 365](/microsoft-365/compliance/advanced-audit).

### <a name="how-is-the-service-provisioneddeployed"></a>Как подготавливается или развертывается служба?

По умолчанию расширенный аудит включен на уровне клиента для всех пользователей, которые получают преимущества от службы, и автоматически предоставляет однолетние журналы аудита для действий (выполняемых пользователями с соответствующей лицензией) в Azure Active Directory, Exchange и SharePoint. Кроме того, организации могут использовать политики хранения журнала аудита для управления периодом хранения записей аудита, созданных действиями в других Microsoft 365 служб. 10-летная функция хранения журнала аудита также включена с помощью тех же политик хранения. Дополнительные сведения см. в статье [Управление политиками хранения журнала аудита](/microsoft-365/compliance/audit-log-retention-policies).

### <a name="how-can-the-service-be-applied-only-to-users-in-the-tenant-who-are-licensed-for-the-service"></a>Как служба может применяться только к пользователям в клиенте, у которых есть лицензия на службу?

Хранение журналов аудита на один год и аудит важных событий применяются только к пользователям с соответствующей лицензией. Кроме того, администраторы могут использовать политики хранения журнала аудита, чтобы указать более короткие сроки хранения для журналов аудита определенных пользователей.

10-летное хранение журналов аудита применяется только к пользователям с соответствующей лицензией на надстройку. Номер SKU надстройки потребуется с начала 2021 г.

## <a name="azure-active-directory-identity-governance"></a>Azure Active Directory управления удостоверениями

Azure Active Directory identity Governance позволяет сбалансировать требования организации к безопасности и производительности сотрудников с правильными процессами и видимостью. Он использует управление правами, проверки доступа, управление привилегированными пользователями и политики условий использования, чтобы обеспечить правильный доступ нужных пользователей к нужным ресурсам.

### <a name="how-do-users-benefit-from-the-service"></a>Как пользователи получают преимущества от этой службы?

Azure Active Directory identity Governance повышает производительность пользователей, упрощая запрос доступа к приложениям, группам и Microsoft Teams в одном пакете доступа. Пользователей также можно настроить в качестве утверждающих без привлечения администраторов. Для проверки доступа пользователи могут просматривать членство в группах с помощью интеллектуальных рекомендаций, чтобы выполнять действия через регулярные интервалы.

### <a name="which-licenses-provide-the-rights-for-a-user-to-benefit-from-the-service"></a>Какие лицензии предоставляют пользователю права на использование службы?

Enterprise Mobility + Security E5/A5, Microsoft 365 E5/A5, Microsoft 365 E5/A5/F5 Security & Compliance, and Azure Active Directory Premium Plan 2 provide the rights for a user to benefit from Azure Active Directory управления удостоверениями.

### <a name="how-is-the-service-provisioneddeployed"></a>Как подготавливается или развертывается служба?

Функции управления удостоверениями Azure AD включены на уровне клиента, но реализуются для каждого пользователя. Сведения об управлении удостоверениями Azure AD см. в статье ["Что такое система управления удостоверениями Azure AD"?](/azure/active-directory/governance/identity-governance-overview)

### <a name="how-can-the-service-be-applied-only-to-users-in-the-tenant-who-are-licensed-for-the-service"></a>Как служба может применяться только к пользователям в клиенте, у которых есть лицензия на службу?

Администраторы могут ограничить область управления удостоверениями Azure AD, назначая пакеты доступа, проверки доступа или управление привилегированными пользователями только лицензированным пользователям. Инструкции по области развертывания Системы управления удостоверениями Azure AD см. в следующих статьях:

- [Требования к лицензии на управление правами Azure AD](/azure/active-directory/governance/entitlement-management-overview#license-requirements)
- [Требования к лицензии на проверку доступа Azure AD](/azure/active-directory/governance/access-reviews-overview#license-requirements)
- [Требования к лицензиям для использования управление привилегированными пользователями](/azure/active-directory/privileged-identity-management/subscription-requirements)

## <a name="azure-active-directory-identity-protection"></a>Защита идентификации Azure Active Directory

Azure Active Directory Identity Protection — это функция плана Azure Active Directory Premium P2, которая позволяет обнаруживать потенциальные уязвимости, влияющие на удостоверения организации, настраивать автоматические ответы на обнаруженные подозрительные действия, связанные с удостоверениями вашей организации, а также исследовать подозрительные инциденты и выполните соответствующие действия, чтобы устранить их.

### <a name="how-do-users-benefit-from-the-service"></a>Как пользователи получают преимущества от этой службы?

Аналитики SecOps и специалисты по безопасности получают преимущества от консолидированных представлений помеченных пользователей и событий риска на основе алгоритмов машинного обучения. Конечные пользователи получают преимущества от автоматической защиты, предоставляемой условным доступом на основе рисков, и улучшенной безопасности, предоставляемой с помощью действий по устранению уязвимостей.

### <a name="which-licenses-provide-the-rights-for-a-user-to-benefit-from-the-service"></a>Какие лицензии предоставляют пользователю права на использование службы?

- Azure Active Directory план 1: Microsoft 365 E3/A3/G3/F1/F3, Enterprise Mobility & Security E3 и Microsoft 365 бизнес премиум
- Azure Active Directory плана 2: Microsoft 365 E5/A5/G5, Enterprise Mobility & Security E5, Microsoft 365 E5/F5 Security and Microsoft 365 F5 Security & Compliance

Дополнительные сведения о возможностях, включенных в различные доступные планы, см. [в Azure Active Directory Identity Protection?](https://www.microsoft.com/en-us/security/business/identity-access-management/azure-ad-pricing)

### <a name="how-is-the-service-provisioneddeployed"></a>Как подготавливается или развертывается служба?

По умолчанию функции защиты идентификации Azure AD включены на уровне клиента для всех пользователей в клиенте. Дополнительные сведения о защите идентификации Azure AD см. [в статье "Что такое защита идентификации"?](/azure/active-directory/identity-protection/overview-identity-protection)

### <a name="how-can-the-service-be-applied-only-to-users-in-the-tenant-who-are-licensed-for-the-service"></a>Как служба может применяться только к пользователям в клиенте, у которых есть лицензия на службу?

Администраторы могут ограничить область защиты идентификации Azure AD, назначив политики рисков, определяющие уровень сброса пароля и разрешив доступ только лицензированным пользователям. Инструкции по настройке и включению политик риска см. в разделе "Настройка и включение политик риска" в службе "Защита идентификации Azure AD ["](/azure/active-directory/identity-protection/howto-sign-in-risk-policy).

## <a name="communication-compliance"></a>Соответствие требованиям к обмену данными

Соответствие требованиям к обмену Microsoft 365 помогает свести к минимуму риски обмена данными, помогая обнаруживать, записывать и принимать меры по исправлению недопустимых сообщений в организации. Вы можете определить определенные политики, которые записывают внутреннюю и внешнюю электронную почту, Microsoft Teams или сторонние сообщения в вашей организации. Рецензенты могут выполнять соответствующие действия по исправлению, чтобы убедиться, что они соответствуют стандартам сообщений вашей организации.

### <a name="how-do-users-benefit-from-the-service"></a>Как пользователи получают преимущества от этой службы?

Специалисты по соответствию требованиям получают преимущества от службы, отслеживая обмен данными организации с помощью политик соответствия требованиям к обмену данными.

### <a name="which-licenses-provide-the-rights-for-a-user-to-benefit-from-the-service"></a>Какие лицензии предоставляют пользователю права на использование службы?

Office 365 E5/A5/G5, Microsoft 365 E5/A5/G5, Microsoft 365 E5/A5/G5/F5 Compliance, Microsoft 365 F5 Security & Compliance and Microsoft 365 E5/A5/G5 Insider Risk Management предоставляют пользователю права на преимущества связи Соответствия.

### <a name="how-is-the-service-provisioneddeployed"></a>Как подготавливается или развертывается служба?

Администраторы и специалисты по соответствию требованиям создают политики соответствия требованиям к обмену данными в Центр соответствия требованиям Microsoft 365. Эти политики определяют, какие сообщения и пользователи подлежат проверке в организации, определяют настраиваемые условия, которым должны соответствовать сообщения, и указывают, кто должен выполнять проверки.

### <a name="how-can-the-service-be-applied-only-to-users-in-the-tenant-who-are-licensed-for-the-service"></a>Как служба может применяться только к пользователям в клиенте, у которых есть лицензия на службу?

Администраторы выбирают определенных пользователей или группы, которые будут включены в политику соответствия требованиям к обмену данными. При выборе группы они также могут выбрать определенных пользователей в группе, чтобы исключить их из политики соответствия требованиям к обмену данными. Дополнительные сведения о политиках соответствия требованиям к обмену данными см. [начало работы с соответствием требованиям к обмену данными в Microsoft 365](/microsoft-365/compliance/communication-compliance-configure).

## <a name="compliance-manager"></a>Диспетчер соответствия требованиям

[Диспетчер соответствия требованиям (Майкрософт](https://compliance.microsoft.com/compliancemanager)[)](/microsoft-365/compliance/microsoft-365-compliance-center) — это компонент Центр соответствия требованиям Microsoft 365, который помогает управлять требованиями к соответствию вашей организации с большей простотой и удобством. Диспетчер соответствия требованиям поможет вам на протяжении всего пути к соответствию требованиям, от составления списка рисков защиты данных до управления сложными моментами реализации элементов управления, получения сведений о нормативных требованиях и сертификатах, а также отправки отчетов аудиторам.

Диспетчер соответствия требованиям помогает упростить соответствие требованиям и снизить риски, предоставляя следующие возможности:

- Предварительно созданные оценки для общих отраслевых и региональных стандартов и нормативных актов, а также пользовательские оценки в соответствии с вашими особыми потребностями в области обеспечения соответствия требованиям.
- Возможности рабочего процесса для эффективного выполнения оценки рисков с помощью одного средства.
- Подробное пошаговое руководство по предлагаемым действиям по улучшению, которые помогут вам соответствовать стандартам и нормативным требованиям, наиболее актуальным для вашей организации. Для действий, управляемых корпорацией Майкрософт, вы увидите сведения о реализации и результаты аудита.
- Оценка соответствия на основе рисков, которая поможет вам понять состояние соответствия требованиям, измеряя ход выполнения действий по улучшению.

### <a name="who-can-access-compliance-manager"></a>Кто может получить доступ к диспетчеру соответствия требованиям?

Диспетчер соответствия требованиям доступен организациям с лицензиями Office 365 и Microsoft 365, а также клиентам из облако сообщества для государственных организаций США (GCC), GCC High и Министерства обороны США (DoD). Доступность оценки и возможности управления зависят от лицензионного соглашения.

### <a name="what-are-premium-assessments"></a>Что такое оценки уровня "Премиум"?

Premium оценки являются дополнительным значением для диспетчера соответствия требованиям и справки:

- Перевод сложных нормативных требований в конкретные элементы управления
- Рекомендуемые действия по улучшению
- Предоставление измеряемой меры соответствия нормативным требованиям

Диспетчер соответствия требованиям содержит более 300 оценок уровня "Премиум", которые клиенты могут использовать для оценки соответствия широкому спектру глобальных, региональных и отраслевых нормативов и стандартов.

Любой клиент с подпиской, которая включает Microsoft Exchange Online лицензию, может приобрести оценки Диспетчера соответствия требованиям уровня "Премиум".

### <a name="what-are-custom-assessments"></a>Что такое пользовательские оценки?

Пользовательские оценки — это функция диспетчера соответствия требованиям, которая позволяет создать новый шаблон или настроить существующий шаблон оценки, включая добавление или обновление элементов управления и действия по улучшению.

- Microsoft 365 A5/E5/F5/G5 Управление внутренними рисками
- Microsoft 365 A5/E5/F5/G5 Information Protection и управление

## <a name="compliance-program-for-microsoft-cloud"></a>Программа соответствия требованиям для Microsoft Cloud

[Программа соответствия требованиям для Microsoft Cloud](https://aka.ms/cpmc) предназначена для предоставления персонализированной поддержки клиентов, образования и сетевых возможностей. Присоединившись к программе, клиенты получат уникальную возможность напрямую взаимодействовать с регулирующими, отраслевыми коллегами и экспертами Майкрософт в области безопасности, соответствия требованиям и конфиденциальности. ## <a name="information-barriers"></a>Информационные барьеры

Информационные барьеры — это политики, которые администратор может настроить, чтобы запретить отдельным пользователям или группам взаимодействовать друг с другом. Это полезно, если, например, один отдел обрабатывает сведения, которые не должны быть общими для других отделов, или если группе необходимо запретить общение с внешними контактами. Политики информационных барьеров также препятствуют поиску и обнаружению. Это означает, что при попытке связаться с пользователем, с которым не следует общаться, вы не найдете этого пользователя в средстве выбора людей.

### <a name="how-do-users-benefit-from-the-service"></a>Как пользователи получают преимущества от этой службы?

Пользователи получают преимущества расширенных возможностей соответствия требованиям информационных барьеров, когда им запрещено взаимодействовать с другими пользователями. Политики информационных барьеров можно определить, чтобы запретить определенному сегменту пользователей взаимодействовать с каждым или разрешить определенным сегментам взаимодействовать только с определенными другими сегментами. Дополнительные сведения об определении политик информационных барьеров см. в разделе ["Определение политик информационных барьеров"](/microsoft-365/compliance/information-barriers-policies). В сценариях, в которых две группы не могут взаимодействовать друг с другом, пользователям в обеих группах требуется лицензия, чтобы воспользоваться преимуществами службы (см. ниже пример).<br><br>

| Сценарий | Кто требуется лицензия? |
|:------|:------|
| Две группы (Group1 и Group2&nbsp;) не могут взаимодействовать друг с другом (т. е. пользователи Group1&nbsp; не могут взаимодействовать с пользователями Group2&nbsp;, а пользователи Group2&nbsp; не могут взаимодействовать с пользователями Group1&nbsp;.&nbsp; | Пользователи в группах&nbsp; 1 и&nbsp; 2 |

Microsoft 365 E5/A5/G5, Microsoft 365 E5/A5/G5/F5 compliance and F5 Security & Compliance, Microsoft 365 E5/A5/G5 Insider Risk Management, and Office 365 E5/A5/G5, provide the rights for a user to benefit from information barriers.

Администраторам необходимо назначить роль Microsoft 365 корпоративный глобального администратора, Office 365 глобального администратора или администратора соответствия требованиям для создания политики информационных барьеров. По умолчанию эти политики применяются ко всем пользователям в клиенте. Дополнительные сведения о информационных барьерах см. в разделе ["Информационные барьеры" Microsoft Teams](/MicrosoftTeams/information-barriers-in-teams).

### <a name="how-can-the-service-be-applied-only-to-users-in-the-tenant-who-are-licensed-for-the-service"></a>Как служба может применяться только к пользователям в клиенте, у которых есть лицензия на службу?

Администраторы могут настраивать расположения (рабочие нагрузки), включенных пользователей и исключенных пользователей в Центре соответствия &amp; требованиям безопасности. Обозреватель действий отображает действия, связанные с конфиденциальными данными и меткой, такие как понижение уровня меток или внешний общий доступ, которые могут привести к риску для содержимого.

Обозреватель действий предоставляет администраторам единую панель для получения сведений о действиях, связанных с конфиденциальной информацией, используемой конечными пользователями. Обозреватель содержимого предоставляет администраторам возможность индексировать конфиденциальные документы, хранящиеся в поддерживаемых рабочих нагрузках Microsoft 365, и определять конфиденциальную информацию, которую они хранят. Кроме того, обозреватель содержимого помогает определить документы, классифицированные с метками конфиденциальности и хранения.

Управление приложениями — это возможность управления безопасностью и политиками, доступная в качестве функции надстройки для Microsoft Defender for Cloud Apps.

Благодаря управлению приложениями клиенты могут отслеживать и контролировать сторонние и собственные приложения на платформе Microsoft 365, чтобы выявлять, оповещений и предотвращать рискованный или неопределенный доступ, права или привилегированное использование данных. Управление приложениями предназначено для приложений с поддержкой OAuth, которые Microsoft 365 данные через [Microsoft API Graph](/graph/use-the-api).

Управление приложениями предоставляет клиентам следующие преимущества функций:

- **Глубокая видимость** [](/microsoft-365/compliance/app-governance-visibility-insights-overview) и аналитические сведения. Более глубокое представление о приложениях, которые получают доступ Microsoft 365 к данным и практическим сведениям о настройке и работе приложения в среде.
- **Управление на основе политик:** упреждающее определение и принудительное применение соответствующего поведения приложений с данными  [,](/microsoft-365/compliance/app-governance-app-policies-overview) пользователями и другими приложениями в соответствии с состоянием безопасности и соответствия требованиям вашей организации для доступа к данным.
- **Комплексное:** [обнаружение и](/microsoft-365/compliance/app-governance-detect-remediate-overview) исправление: обнаружение аномального поведения приложения с помощью моделей машинного обучения, устранение проблем с автоматическими действиями по исправлению и действия по исправлению вручную.

Дополнительные сведения см. в разделе "Управление приложениями[" Microsoft 365](/microsoft-365/compliance/app-governance-manage-app-governance) и [Начало работы с управлением приложениями](/microsoft-365/compliance/app-governance-get-started).

### <a name="is-it-a-requirement-for-the-apps-in-the-tenant-to-be-registered-with-azure-active-director-to-be-viewable-by-app-governance"></a>Требуется ли, чтобы приложения в клиенте были зарегистрированы в Azure Active Director, чтобы их можно было просматривать в системе управления приложениями?

Microsoft Defender для удостоверений (ранее — Расширенная защита от угроз Azure) — это облачная служба, которая помогает защитить корпоративные гибридные среды от различных типов сложных целевых киберагром и внутренних угроз.

Аналитики SecOp и специалисты по безопасности получают преимущества от Microsoft Defender для удостоверений обнаружения и исследования сложных угроз, скомпрометированных удостоверений и вредоносных действий внутри организации. ### <a name="how-can-the-service-be-applied-only-to-users-in-the-tenant-who-are-licensed-for-the-service"></a>Как служба может применяться только к пользователям в клиенте, у которых есть лицензия на службу?

Microsoft Defender для удостоверений службы в настоящее время не могут ограничить возможности определенными пользователями. Аналогичным образом, нет требования к лицензии учетных записей служб или учетной записи, которая упрощает автоматизацию. Лицензия должна быть только у пользователей-пользователей.

## <a name="microsoft-priva"></a>Microsoft Priva

[Microsoft Priva](https://aka.ms/privacymanagementdocs) помогает компаниям защитить персональные данные и создать устойчивое к конфиденциальности рабочее место, упреждая выявление и защиту от рисков конфиденциальности, таких как сбор данных, передача данных и общий доступ к данным, предоставление информационным работникам возможности принимать интеллектуальные решения по обработке данных, а также автоматизация запросов субъектов и управление ими в большом масштабе.

### <a name="which-licenses-provide-the-rights-for-a-user-to-benefit-from-the-service"></a>Какие лицензии предоставляют пользователю права на использование службы?

Priva доступна в дополнение к организациям с подписками Office 365 A1/E1/A3/E3/A5/E5 и Microsoft 365 A3/E3/A5/E5. 

### <a name="how-do-users-benefit-from-the-service"></a>Как пользователи получают преимущества от этой службы?

Пользователи получают преимущества благодаря возможности организации получать представление о частных данных в своей среде, заблаговременно выявлять и защищать от рисков конфиденциальности, а также управлять запросами прав субъектов (обычно называемыми запросами субъектов данных) в большом масштабе.

### <a name="how-can-customers-access-the-service"></a>Как клиенты могут получить доступ к службе?

Решения Priva встроены [в Центр соответствия требованиям Microsoft 365 и](https://compliance.microsoft.com/homepage) включены на уровне клиента для всех пользователей в клиенте. Мы рекомендуем получать лицензии для любого пользователя, которого вы планируете использовать и защитить с помощью службы.

Клиенты могут приобрести следующие лицензии в соответствии с потребностями организации:

**Управление конфиденциальностью — это имя лицензии Priva Privacy Risk Management.** Служба позволяет организациям выполнять следующие действия:

- Получите представление о персональных данных в среде Microsoft 365 (Microsoft Exchange Online, SharePoint, OneDrive для бизнеса и Teams) и связанных с ними рисках.
- Используйте шаблоны политики конфиденциальности по умолчанию, включая минимизацию данных, чрезмерное использование данных и передачу данных, или настройте их в соответствии с уникальными потребностями организации.
- Получите рекомендуемые средства управления исправлениями для снижения рисков конфиденциальности.
- Взаимодействие с информационными работниками из набора средств повышения производительности и изменение поведения.

**Управление конфиденциальностью — запросы прав субъекта — это имя лицензии priva Subject Rights Requests.** Служба позволяет организациям выполнять следующие действия:

- Автоматизация ответа на запросы прав субъекта и управление ими в большом масштабе.
- Используйте шаблоны microsoft Power Automate с существующими бизнес-процессами (для Power Automate) потребуется соответствующая лицензия.
- Использование программного доступа к API.
- Безопасная совместная работа с другими заинтересованными Microsoft Teams интеграции (потребуется соответствующая лицензия для Microsoft Teams).</br>

Клиенты смогут приобрести запросы прав субъекта Priva в блоках 1,10 или 100.

Управление рисками конфиденциальности и запросы прав субъекта Priva можно приобрести независимо друг от друга.

Ознакомьтесь [с условиями лицензионного](https://www.microsoft.com/licensing/terms/productoffering/Microsoft365/EAEAS) соглашения, необходимыми для получения запросов на управление рисками конфиденциальности и правами субъекта Priva.

## <a name="office-365-customer-lockbox"></a>Защищенное хранилище в Office 365

Защищенное хранилище клиентов предоставляет дополнительный уровень контроля, предлагая клиентам возможность явно предоставлять авторизацию доступа для операций службы. Демонстрация того, что процедуры для явной авторизации доступа к данным выполняются, защищенное хранилище также может помочь организациям выполнить определенные обязательства по соответствию, такие как HIPAA и FedRAMP.

### <a name="how-do-users-benefit-from-the-service"></a>Как пользователи получают преимущества от этой службы?

Защищенное хранилище гарантирует, что никто в корпорации Майкрософт не сможет получить доступ к содержимому клиента для выполнения операции службы без явного утверждения клиента. Защищенное хранилище позволяет клиенту в рабочем процессе утверждения запрашивать доступ к его содержимому. Иногда инженеры Майкрософт участвуют в процессе поддержки для устранения неполадок и устранения проблем, о которых сообщили клиенты. В большинстве случаев проблемы устранены с помощью обширных средств телеметрии и отладки, которые корпорация Майкрософт использует для своих служб. Однако в некоторых случаях инженеру Майкрософт требуется доступ к содержимому клиента для определения первопричины и устранения проблемы. Если используется защищенное хранилище, инженер должен запросить у клиента доступ на последнем этапе рабочего процесса утверждения. Это дает организациям возможность утверждать или отклонять эти запросы, что дает им прямой контроль над доступом инженера Майкрософт к данным конечных пользователей организации.

### <a name="which-licenses-provide-the-rights-for-a-user-to-benefit-from-the-service"></a>Какие лицензии предоставляют пользователю права на использование службы?

Office 365 E5/A5/G5, Microsoft 365 E5/A5/G5, Microsoft 365 E5/A5/G5/F5 Compliance and F5 Security & Compliance, and Microsoft 365 E5/A5/G5 Insider Risk Management provide the rights for a user to benefit from Customer Lockbox.

### <a name="how-is-the-service-provisioneddeployed"></a>Как подготавливается или развертывается служба?

Администраторы могут включить защищенное хранилище клиента в Центр администрирования Microsoft 365. Дополнительные сведения см[. в разделе "Защищенное хранилище клиента" Office 365](/microsoft-365/compliance/customer-lockbox-requests). Если защищенное хранилище клиента включено, корпорация Майкрософт должна получить утверждение организации перед доступом к любому из их содержимого.

### <a name="how-can-the-service-be-applied-only-to-users-in-the-tenant-who-are-licensed-for-the-service"></a>Как служба может применяться только к пользователям в клиенте, у которых есть лицензия на службу?

В настоящее время служба защищенного хранилища клиентов не может быть ограничена определенными пользователями. Хотя службы клиента в настоящее время не могут ограничивать преимущества определенными пользователями, необходимо предпринять меры, чтобы ограничить преимущества службы лицензированными пользователями. Это поможет избежать потенциального сбоя службы после того, как будут доступны целевые возможности.

### <a name="how-do-users-benefit-from-the-service"></a>Как пользователи получают преимущества от этой службы?

Пользователи могут объявлять содержимое как запись и управлять процессом полных записей из определения политики и объявления посредством defensible disposal.

### <a name="how-is-the-service-provisioneddeployed"></a>Как подготавливается или развертывается служба?

По умолчанию функции управления записями включены на уровне клиента для всех пользователей в клиенте. Сведения о настройке управления записями для лицензированных пользователей см. в разделе "Управление [записями"](/microsoft-365/compliance/records-management) Microsoft 365.

### <a name="how-can-the-service-be-applied-only-to-users-in-the-tenant-who-are-licensed-for-the-service"></a>Как служба может применяться только к пользователям в клиенте, у которых есть лицензия на службу?

Функции управления записями можно применять к лицензированным пользователям в определенных расположениях (сайты групп, сайты групп и т. д.). Сведения о настройке управления записями для лицензированных пользователей см. в разделе "Управление [записями"](/microsoft-365/compliance/records-management) Microsoft 365.
