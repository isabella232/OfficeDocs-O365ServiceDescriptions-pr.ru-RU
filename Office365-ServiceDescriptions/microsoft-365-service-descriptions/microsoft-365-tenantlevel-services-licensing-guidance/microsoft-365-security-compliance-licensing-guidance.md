# <a name="microsoft-365-guidance-for-security-amp-compliance"></a>Microsoft 365 инструкции по обеспечению соответствия требованиям &amp; безопасности

Для целей этой статьи служба на уровне клиента — это служба, активированная частично или полностью для всех пользователей в клиенте (автономные лицензии и(или) в рамках Microsoft 365 или Office 365 плана). Необходимы соответствующие лицензии на подписку для использования клиентами онлайн-служб. Чтобы узнать о вариантах лицензирования пользователей, Microsoft 365 параметров соответствия требованиям, скачайте таблицу Microsoft 365  [сравнение](https://go.microsoft.com/fwlink/?linkid=2139145).

Некоторые службы клиентов в настоящее время не могут ограничивать преимущества для определенных пользователей. Необходимо принять меры для ограничения преимуществ службы для лицензированных пользователей. Обзор условий и условий, регулирующих использование продуктов и Professional служб Майкрософт, приобретенных с помощью программ лицензирования Майкрософт, см. в обзоре  [ТерминыProduct](https://www.microsoft.com/Licensing/product-licensing/products).


## <a name="advanced-audit"></a>Расширенный аудит

Расширенный аудит в Microsoft 365 предоставляет однолетнее хранение журналов аудита для действий пользователя и администратора и предоставляет возможность создавать настраиваемые политики хранения журналов аудита для управления хранением журналов аудита для других Microsoft 365 служб. Он также предоставляет доступ к важным событиям для расследований и высокой пропускной способности к API Office 365 управления. Дополнительные сведения см. в [дополнительных сведениях](/microsoft-365/compliance/advanced-audit) в Microsoft 365.

Кроме того, можно включить срок хранения в 10 лет с помощью надстройки SKU.

### <a name="which-users-benefit-from-the-service"></a>Какие пользователи получают выгоду от службы?

Лицензированные пользователи Office 365 E5/A5/G5, Microsoft 365 E5/A5/G5, Microsoft 365 E5/A5/G5/F5 Compliance, Microsoft 365 F5 Security & Compliance и Microsoft 365 E5/A5/G5 eDiscovery and Audit могут воспользоваться расширенным аудитом.

Лицензированные пользователи с расширенным аудитом и надстройка хранения журналов аудита сроком на 10 лет могут воспользоваться 10-летней возможностью хранения журналов аудита.

### <a name="how-do-users-benefit-from-the-service"></a>Как пользователи получают выгоду от службы?

Расширенный аудит выгоден пользователям, так как записи аудита, связанные с активностью пользователей в Microsoft 365 службах, могут храниться до одного года. Кроме того, регистрируются события аудита с высокой ценностью, например при доступе или прочтение элементов в почтовом ящике пользователя. Дополнительные сведения см. в [дополнительных сведениях](/microsoft-365/compliance/advanced-audit) в Microsoft 365.

### <a name="how-is-the-service-provisioneddeployed"></a>Как подготовка или развертывание службы?

По умолчанию расширенный аудит включен на уровне клиента для всех пользователей, которые получают выгоду от службы, и автоматически предоставляет однолетнее хранение журналов аудита для действий (выполняемые пользователями с соответствующей лицензией) в Azure Active Directory, Exchange и SharePoint. Кроме того, организации могут использовать политики хранения журналов аудита для управления периодом хранения записей аудита, созданных в результате действий в других Microsoft 365 службах. 10-летняя функция хранения журналов аудита также включена с помощью тех же политик хранения. Дополнительные сведения см. в статье [Управление политиками хранения журнала аудита](/microsoft-365/compliance/audit-log-retention-policies).

### <a name="how-can-the-service-be-applied-only-to-users-in-the-tenant-who-are-licensed-for-the-service"></a>Как можно применять службу только к пользователям в клиенте, которые имеют лицензию на службу?

Однолетнее хранение журналов аудита и аудит важных событий применяются только к пользователям с соответствующей лицензией. Кроме того, администраторы могут использовать политики хранения журналов аудита, чтобы указать более короткие сроки хранения журналов аудита определенных пользователей.

10-летнее хранение журналов аудита применяется только к пользователям с соответствующей лицензией надстройки. Надстройка SKU потребуется с начала 2021 г.

## <a name="azure-active-directory-identity-protection"></a>Защита идентификации Azure Active Directory

Azure Active Directory Защита удостоверений — это функция плана Azure Active Directory Premium P2, который позволяет обнаруживать потенциальные уязвимости, влияющие на удостоверения организации, настраивать автоматические ответы на обнаруженные подозрительные действия, связанные с удостоверениями организации, а также расследовать подозрительные инциденты и принять соответствующие меры для их устранения.

### <a name="how-do-users-benefit-from-the-service"></a>Как пользователи получают выгоду от службы?

Аналитики и специалисты по безопасности SecOps извлекли выгоду из консолидированных представлений о помеченных пользователях и событиях риска на основе алгоритмов машинного обучения. Конечные пользователи пользуются автоматической защитой, предоставляемой с помощью условного доступа на основе риска, и улучшенной безопасностью, предоставляемой с помощью защиты от уязвимостей.

### <a name="which-licenses-provide-the-rights-for-a-user-to-benefit-from-the-service"></a>Какие лицензии предоставляют пользователю права на выгоду от службы?

- Azure Active Directory 1: Microsoft 365 E3/A3/G3/F1/F3, Enterprise Mobility & Security E3 и Microsoft 365 бизнес премиум
- Azure Active Directory 2: Microsoft 365 E5/A5/G5, Enterprise Mobility & Security E5, Microsoft 365 E5/F5 Security и Microsoft 365 F5 Security & Compliance

Сведения о возможностях, включенных в различные доступные планы, см. в Azure Active Directory [Identity Protection?](/azure/active-directory/identity-protection/overview-identity-protection#license-requirements)

### <a name="how-is-the-service-provisioneddeployed"></a>Как подготовка или развертывание службы?

По умолчанию функции Azure AD Identity Protection включены на уровне клиента для всех пользователей в клиенте. Сведения о защите удостоверений Azure AD см. в видеоролике [Что такое защита удостоверений?](/azure/active-directory/identity-protection/overview-identity-protection)

### <a name="how-can-the-service-be-applied-only-to-users-in-the-tenant-who-are-licensed-for-the-service"></a>Как можно применять службу только к пользователям в клиенте, которые имеют лицензию на службу?

Администраторы могут расширить область Azure AD Identity Protection, назначив политики риска, определяющие уровень сброса паролей и разрешающие доступ только лицензированным пользователям. Инструкции по развертыванию azure AD Identity Protection см. в инструкции по настройке и [встройке политик риска](/azure/active-directory/identity-protection/howto-sign-in-risk-policy).

## <a name="azure-active-directory-identity-governance"></a>Azure Active Directory управления удостоверением

Azure Active Directory управление удостоверениями позволяет уравновешить потребности организации в безопасности и производительности сотрудников с нужными процессами и наглядностью. Он использует правила управления правами, обзоры доступа, привилегированное управление удостоверениями и политики использования терминов для обеспечения правильного доступа нужных людей к нужным ресурсам.

### <a name="how-do-users-benefit-from-the-service"></a>Как пользователи получают выгоду от службы?

Azure Active Directory управления удостоверениями повышает производительность пользователей, упрощая запрос доступа к приложениям, группам и Microsoft Teams одному пакету доступа. Пользователи также могут быть настроены в качестве одобрений, без привлечения администраторов. Для отзывов о доступе пользователи могут просмотреть членство групп с интеллектуальными рекомендациями, чтобы действовать регулярно.

### <a name="which-licenses-provide-the-rights-for-a-user-to-benefit-from-the-service"></a>Какие лицензии предоставляют пользователю права на выгоду от службы?

Enterprise Mobility + Security E5/A5, Microsoft 365 E5/A5, Microsoft 365 E5/A5/F5 Security & Compliance и Azure Active Directory Premium Plan 2 предоставляют пользователям права на выгоду от Azure Active Directory управления удостоверением.

### <a name="how-is-the-service-provisioneddeployed"></a>Как подготовка или развертывание службы?

Функции управления удостоверением Azure AD включены на уровне клиента, но реализуются для каждого пользователя. Сведения об управлении удостоверением Azure AD см. в см. в нем [. Что такое управление удостоверением Azure AD?](/azure/active-directory/governance/identity-governance-overview)

### <a name="how-can-the-service-be-applied-only-to-users-in-the-tenant-who-are-licensed-for-the-service"></a>Как можно применять службу только к пользователям в клиенте, которые имеют лицензию на службу?

Администраторы могут расширить область управления удостоверениями Azure AD, назначив пакеты доступа, обзоры доступа или привилегированное управление удостоверениями только для лицензированных пользователей. Инструкции по развертыванию служб управления удостоверениями Azure AD см. в этой инструкции:

- [Требования к лицензии на управление правами Azure AD](/azure/active-directory/governance/entitlement-management-overview#license-requirements)
- [Требования лицензии на проверку доступа к Azure AD](/azure/active-directory/governance/access-reviews-overview#license-requirements)
- [Требования лицензии для использования управление привилегированными пользователями](/azure/active-directory/privileged-identity-management/subscription-requirements)

## <a name="communication-compliance"></a>Соответствие требованиям к обмену данными

Соответствие требованиям к Microsoft 365 позволяет свести к минимуму риски для связи, помогая обнаруживать, захватывать и принимать меры по исправлению неподобающего сообщения в организации. Можно определить определенные политики, которые захватывают внутреннюю и внешнюю электронную почту, Microsoft Teams или сторонние сообщения в вашей организации. Проверяющие могут принимать соответствующие меры по исправлению, чтобы убедиться, что они соответствуют стандартам сообщений организации.

### <a name="how-do-users-benefit-from-the-service"></a>Как пользователи получают выгоду от службы?

Специалисты по обеспечению соответствия требованиям получают преимущество от службы, контролируя связи организации политиками соответствия требованиям.

### <a name="which-licenses-provide-the-rights-for-a-user-to-benefit-from-the-service"></a>Какие лицензии предоставляют пользователю права на выгоду от службы?

Office 365 E5/A5/G5, Microsoft 365 E5/A5/G5, Microsoft 365 E5/A5/G5/F5 Compliance, Microsoft 365 F5 Security & Compliance and Microsoft 365 E5/A5/G5 Insider Risk Management provide the rights for a user to benefit from communication соответствие требованиям.

### <a name="how-is-the-service-provisioneddeployed"></a>Как подготовка или развертывание службы?

Администраторы и специалисты по обеспечению соответствия требованиям создают политики соответствия Центр соответствия требованиям Microsoft 365. Эти политики определяют, какие сообщения и пользователи подлежат проверке в организации, определяют настраиваемые условия, которые должны соответствовать коммуникациям, и определяют, кто должен выполнять обзоры.

### <a name="how-can-the-service-be-applied-only-to-users-in-the-tenant-who-are-licensed-for-the-service"></a>Как можно применять службу только к пользователям в клиенте, которые имеют лицензию на службу?

Администраторы выбирают определенных пользователей или групп, чтобы включить их в политику соответствия требованиям. При выборе группы они также могут выбрать определенных пользователей в группе, чтобы исключить их из политики соответствия требованиям. Дополнительные сведения о политиках соответствия требованиям к коммуникациям см. в этой ссылке[: Начало работы с](/microsoft-365/compliance/communication-compliance-configure) соблюдением требований к Microsoft 365.

## <a name="compliance-manager"></a>Диспетчер соответствия требованиям

[Microsoft Compliance Manager](https://compliance.microsoft.com/compliancemanager) — это функция в Центр соответствия требованиям Microsoft 365[](/microsoft-365/compliance/microsoft-365-compliance-center), которая позволяет с большей легкостью и удобством управлять требованиями соответствия требованиям организации. Диспетчер соответствия требованиям поможет вам на протяжении всего пути к соответствию требованиям, от составления списка рисков защиты данных до управления сложными моментами реализации элементов управления, получения сведений о нормативных требованиях и сертификатах, а также отправки отчетов аудиторам.

Диспетчер соответствия требованиям помогает упростить соответствие требованиям и снизить риск, предоставив:

- Предварительно созданные оценки для общих отраслевых и региональных стандартов и нормативных актов, а также пользовательские оценки в соответствии с вашими особыми потребностями в области обеспечения соответствия требованиям.
- Возможности рабочего процесса для эффективного выполнения оценки рисков с помощью одного средства.
- Подробные пошаговая инструкция по предлагаемым действиям по улучшению, которые помогут вам соответствовать стандартам и правилам, наиболее актуальным для вашей организации. Для действий, управляемых Корпорацией Майкрософт, вы увидите сведения о реализации и результаты аудита.
- Оценка соответствия требованиям, основанным на рисках, которая поможет вам понять вашу позицию соответствия требованиям, измеряя ваши успехи в выполнении действий по улучшению.

### <a name="who-can-access-compliance-manager"></a>Кто может получить доступ к диспетчеру соответствия требованиям?

Диспетчер соответствия требованиям доступен организациям с Office 365 и Microsoft 365 лицензиями, а также клиентам облако сообщества для государственных организаций (GCC), GCC High и Department of Defense (DoD). Доступность оценки и возможности управления зависят от лицензионного соглашения.

### <a name="what-are-premium-assessments"></a>Что такое оценки премиум-класса?

Premium оценки являются дополнительным значением для диспетчера соответствия требованиям и помогают:

- Перевод сложных нормативных требований на конкретные элементы управления
- Рекомендуемые действия по улучшению
- Предоставление измеряемой меры соответствия требованиям нормативных положений

Диспетчер соответствия требованиям имеет более 300 оценок премиум-класса, которые клиенты могут использовать для оценки соответствия широкому кругу глобальных, региональных и промышленных правил и стандартов.

Любой клиент с подпиской, которая включает Microsoft Exchange Online лицензию, может приобрести оценки премиум-класса Compliance Manager.

### <a name="which-premium-assessments-are-available"></a>Какие оценки премиум-класса доступны?

Вот список [оценок премиум-класса](/microsoft-365/compliance/compliance-manager-templates-list#premium-templates).

### <a name="what-are-custom-assessments"></a>Что такое настраиваемые оценки?

Настраиваемые оценки — это функция Диспетчер соответствия требованиям, которая предоставляет возможность создания нового шаблона или настройки существующего шаблона оценки, включая добавление или обновление элементов управления и действий по улучшению.

### <a name="who-can-access-custom-assessments"></a>Кто можно получить доступ к пользовательским оценкам?

Функция настраиваемой оценки доступна клиентам с подпиской на E5, как указано ниже:

- Microsoft 365 или Office 365 A5/E5/G5
- Microsoft 365 A5/E5/F5/G5 Compliance
- Microsoft 365 A5/E5/F5/G5 eDiscovery and Audit
- Microsoft 365 A5/E5/F5/G5 Insider Risk Management
- Microsoft 365 A5/E5/F5/G5 Защита и управление информацией

## <a name="compliance-program-for-microsoft-cloud"></a>Программа соответствия требованиям для Microsoft Cloud

[Программа соответствия требованиям для Microsoft Cloud](https://aka.ms/cpmc) предназначена для обеспечения персонализированной поддержки клиентов, образования и сетевых возможностей. Присоединившись к программе, клиенты получат уникальную возможность напрямую взаимодействовать с регулирующими, отраслевыми коллегами и экспертами Майкрософт в области безопасности, соответствия требованиям и конфиденциальности. Эта программа заменяет существующую программу соответствия требованиям отрасли финансовых услуг (FSI), созданную в 2013 году.

### <a name="who-can-access-the-compliance-program-for-microsoft-cloud"></a>Кто можно получить доступ к программе соответствия требованиям для Microsoft Cloud?

## <a name="information-protection-office-365-advanced-message-encryption"></a>Защита информации: Расширенное шифрование сообщений Office 365

Расширенное шифрование сообщений Office 365 помогает клиентам выполнять обязательства по обеспечению соответствия требованиям, которые требуют более гибкого контроля над внешними получателями и доступом к зашифрованным электронным письмам. С помощью расширенного шифрования сообщений администраторы могут управлять конфиденциальными электронными сообщениями, общими за пределами организации, с помощью автоматических политик, которые могут обнаруживать типы конфиденциальной информации (например, персональные идентификации информации или финансовые или медицинские идентификации), или они могут использовать ключевые слова для повышения защиты, применяя настраиваемые шаблоны электронной почты и истекая доступ к зашифрованным электронным письмам через защищенный веб-портал. Кроме того, администраторы могут дополнительно управлять зашифрованными электронными письмами, к которых можно получить доступ извне через безопасный веб-портал, отменив доступ в любое время.

## <a name="information-barriers"></a>Информационные барьеры

Информационные барьеры — это политики, которые администратор может настроить, чтобы запретить отдельным лицам или группам общаться друг с другом. Это полезно, если, например, один отдел занимается обработкой сведений, которые не следует делиться с другими отделами, или группе необходимо запретить общаться с внешними контактами. Политики информационного барьера также препятствуют обнаружению и обнаружению. Это означает, что если вы попытаетесь связаться с кем-то, с кем не следует общаться, этот пользователь не будет находиться в средстве поиска людей.

### <a name="how-do-users-benefit-from-the-service"></a>Как пользователи получают выгоду от службы?

Пользователи получают выгоду от расширенных возможностей обеспечения соответствия требованиям информационных барьеров, когда им запрещено общаться с другими пользователями. Политики информационных барьеров можно определить, чтобы предотвратить связь определенных сегментов пользователей с каждым или разрешить определенным сегментам общаться только с определенными другими сегментами. Дополнительные сведения о определении политик информационного барьера см. в дополнительных сведениях [о политиках информационного барьера](/microsoft-365/compliance/information-barriers-policies). Для сценариев, в которых две группы не могут общаться друг с другом, пользователям обеих групп требуется лицензия для получения выгоды от службы (см. ниже пример).<br><br>

Администраторы создают и управляют политиками информационного барьера с помощью cmdlets PowerShell в &amp; Центре соответствия требованиям безопасности. Администраторам необходимо награждение Microsoft 365 корпоративный глобального администратора, Office 365 глобального администратора или администратора соответствия требованиям для создания политики информационного барьера. По умолчанию эти политики применяются к всем пользователям в клиенте. Дополнительные сведения о информационных барьерах см. [в Microsoft Teams.](/MicrosoftTeams/information-barriers-in-teams)

### <a name="how-can-the-service-be-applied-only-to-users-in-the-tenant-who-are-licensed-for-the-service"></a>Как можно применять службу только к пользователям в клиенте, которые имеют лицензию на службу?

Администраторы могут настраивать расположения (рабочие нагрузки), включаемые пользователи и исключенные пользователи в &amp; Центре соответствия требованиям безопасности. Например, если все пользователи имеют лицензии на Office 365 E3 и ни один из них не лицензирован для Office 365 Advanced Compliance/E5, им не потребуется создавать политики информационного барьера для организации. ## <a name="insider-risk-management"></a>Управление внутренними рисками

## <a name="office-365-customer-lockbox"></a>Защищенное хранилище в Office 365

Lockbox клиента предоставляет дополнительный уровень управления, предлагая клиентам возможность предоставления явной авторизации доступа для операций службы. Демонстрируя, что для явного разрешения доступа к данным есть процедуры, клиент lockbox также может помочь организациям выполнить определенные обязательства по соблюдению, такие как HIPAA и FedRAMP.

### <a name="how-do-users-benefit-from-the-service"></a>Как пользователи получают выгоду от службы?

Lockbox клиента гарантирует, что никто в Корпорации Майкрософт не может получить доступ к содержимому клиента для выполнения операции службы без явного утверждения клиента. Lockbox клиента приводит клиента в рабочий процесс утверждения запросов для доступа к их содержимому. Иногда инженеры Майкрософт участвуют в процессе поддержки для устранения неполадок и устранения проблем, о чем сообщили клиенты. В большинстве случаев проблемы устранены с помощью обширных средств телеметрии и отладки, которые корпорация Майкрософт имеет для своих служб. Однако могут быть случаи, когда инженеру Корпорации Майкрософт требуется доступ к содержимому клиента для определения первопричины и устранения проблемы. Если используется защищенное хранилище, инженер должен запросить у клиента доступ на последнем этапе рабочего процесса утверждения. Это дает организациям возможность утверждать или отказывать в этих запросах, что дает им прямой контроль над тем, может ли инженер Майкрософт получать доступ к данным конечных пользователей организации.

### <a name="which-licenses-provide-the-rights-for-a-user-to-benefit-from-the-service"></a>Какие лицензии предоставляют пользователю права на выгоду от службы?

Office 365 E5/A5/G5, Microsoft 365 E5/A5/G5, Microsoft 365 E5/A5/G5/F5 Compliance and F5 Security & Compliance, and Microsoft 365 E5/A5/G5 Insider Risk Management provide the rights for a user to benefit from customer Lockbox.

### <a name="how-is-the-service-provisioneddeployed"></a>Как подготовка или развертывание службы?

Администраторы могут включить блокировку клиента в Центр администрирования Microsoft 365. Дополнительные сведения см. в [Office 365](/microsoft-365/compliance/customer-lockbox-requests). При включенной блокировке клиента корпорация Майкрософт должна получить одобрение организации перед доступом к любому их содержимому.

### <a name="how-can-the-service-be-applied-only-to-users-in-the-tenant-who-are-licensed-for-the-service"></a>Как можно применять службу только к пользователям в клиенте, которые имеют лицензию на службу?

В настоящее время служба блокировки клиентов не может ограничиваться конкретными пользователями. Несмотря на то, что в настоящее время службы клиента не могут ограничивать преимущества для определенных пользователей, необходимо принять меры для ограничения преимуществ службы для лицензированных пользователей. Это поможет избежать потенциальных сбоев в работе службы после того, как будут доступны возможности таргетинга.

## <a name="microsoft-priva"></a>Microsoft Priva

[Microsoft Priva](https://aka.ms/privacymanagementdocs) помогает компаниям защищать персональные данные и создавать рабочие места, устойчивые к конфиденциальности, активно выявляя и защищая от рисков конфиденциальности, таких как сбор данных, передача данных и перешаговка данных, предоставление сотрудникам информации возможности принимать разумные решения по обработке данных, а также автоматизировать и управлять запросами субъектов в масштабе.

### <a name="which-licenses-provide-the-rights-for-a-user-to-benefit-from-the-service"></a>Какие лицензии предоставляют пользователю права на выгоду от службы?

Priva доступна в качестве дополнения для организаций с подписками Office 365 A1/E1/A3/E3/A5/E5 и Microsoft 365 A3/E3/A5/E5. 

### <a name="how-do-users-benefit-from-the-service"></a>Как пользователи получают выгоду от службы?

Пользователи получают выгоду от возможности для организаций получать доступ к частным данным в своей среде, активно выявлять и защищать от рисков конфиденциальности и управлять запросами на права субъекта (обычно известные как "запросы субъекта данных").

### <a name="how-can-customers-access-the-service"></a>Как клиенты могут получить доступ к службе?

Решения Priva встроены [в Центр соответствия требованиям Microsoft 365 и](https://compliance.microsoft.com/homepage) включены на уровне клиента для всех пользователей в клиенте. Мы рекомендуем приобретать лицензии любому пользователю, который будет получать выгоду от службы и защищать ее.

Клиенты могут приобрести следующие лицензии в зависимости от организационных необходимости:

**Управление конфиденциальностью — это название лицензии управления рисками конфиденциальности Привы.** Служба позволяет организациям:

- Получение видимости персональных данных в среде Microsoft 365 (Microsoft Exchange Online, SharePoint, OneDrive для бизнеса и Teams) и связанных с этим рисков.
- Использование шаблонов политики конфиденциальности по умолчанию, включая минимизацию данных, переэкспорацию данных и передачу данных или настройку их для удовлетворения ваших уникальных организационных потребностей.
- Получение рекомендуемых элементов управления исправлением для снижения рисков конфиденциальности.
- Взаимодействовать с сотрудниками по работе с информацией из набора производительности и управлять изменением поведения.

**Управление конфиденциальностью — запросы на права субъекта — это имя лицензии запросов на права субъекта Priva.** Служба позволяет организациям:

- Автоматизация ответа на запросы о правах субъекта и управление ими в масштабе.
- Используйте шаблоны microsoft Power Automate с существующими бизнес-процессами (потребуется соответствующая лицензия для Power Automate).
- Использование программного доступа к API.
- Безопасно сотрудничать с другими заинтересованными сторонами Microsoft Teams интеграции (потребуется соответствующая лицензия для Microsoft Teams).</br>

Клиенты смогут приобрести запросы на права субъекта Priva в блоках 1,10 или 100.

Запросы на управление рисками конфиденциальности и запросы на права субъекта Привывы можно приобрести независимо друг от друга.

См. [условия продукта для](https://www.microsoft.com/licensing/terms/productoffering/Microsoft365/EAEAS) необходимых условий лицензирования, необходимых для получения запросов на управление рисками конфиденциальности и приватных прав субъектов.
