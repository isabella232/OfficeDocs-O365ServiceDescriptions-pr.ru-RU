# <a name="exchange-online-archiving-service-description"></a>Описание службы архивации на базе Exchange Online

Microsoft Exchange Online Архивативная Microsoft 365 облачное решение для организаций, которые развернули Microsoft Exchange Server 2019, Microsoft Exchange Server 2016 Microsoft Exchange Server 2013 г., или подписаться на определенные  Exchange Online или Microsoft 365 планы. Exchange Online Archiving помогает организациям в выполнении задач архива, соответствия требованиям, нормативных и электронных обнаружений.

Являясь одной из веб-служб Майкрософт, Exchange Online Archiving предназначена помочь организациям повысить безопасность и надежность работы, а также производительность труда. Дополнительные сведения о Microsoft 365, включая функции, общие для всех онлайн-служб Майкрософт, см. в Microsoft 365 и [Office 365 платформы.](../office-365-platform-service-description/office-365-platform-service-description.md)

### <a name="requirements"></a>Требования

Чтобы использовать Exchange Online Archiving для Exchange Server, почтовые ящики пользователей должны находиться Exchange Server 2019, Exchange Server 2016 или Exchange Server 2013 года.

### <a name="user-subscriptions"></a>Подписки пользователей

Каждый пользователь, имеющий доступ к Архивация на базе Exchange Online, должен иметь подписку Архивация на базе Exchange Online. Каждая подписка на архив электронной почты может использоваться для хранения данных сообщений только одного пользователя.

### <a name="federated-identity-and-single-sign-on"></a>Федеративное удостоверение и единый вход

Администраторы могут использовать единый входной подход к проверке подлинности с помощью локального Active Directory. Для этого администраторы могут настроить локальное обслуживание Федерации Active Directory — службы Microsoft Windows Server 2008 для федерации &reg; с помощью Microsoft Federation Gateway. После настройки служб Федерации Active Directory все пользователи, удостоверения которых основаны на федеративном домене, могут использовать существующий корпоративный логотип для автоматической проверки подлинности для Office 365.

### <a name="archive-storage-quota"></a>Квота хранения архива

Администраторы не могут настраивать квоту хранилища. Функция архивирования (так называемая автоматическое расширение архивирования) предоставляет дополнительное пространство для хранения в архивных почтовых ящиках, не более 1,5 ТБ. Каждый подписчик Exchange Online Archiving изначально получает хранилище объемом 100 ГБ в архивном почтовом ящике. При автоматическом расширении архивирования автоматически добавляется дополнительное пространство для хранения, когда достигается емкость хранилища в 100 ГБ. В Exchange гибридных развертываниях автоматическое расширение архивации поддерживается только для облачных архивных почтовых ящиков, когда почтовый ящик локального пользователя находится в Exchange Server 2019 г., Exchange Server 2016 г. или Exchange Server 2013 г. и далее.

Архив автоматического расширения поддерживается только для почтовых ящиков, используемых для отдельных пользователей или общих почтовых ящиков со скоростью роста не более *1 ГБ в день.* Не разрешается использовать функцию ведения журнала, правила транспорта и правила автоматической пересылки, чтобы копировать сообщения на Архивация на базе Exchange Online для создания архива. Архивный почтовый ящик пользователя предназначен только для этого пользователя. Корпорация Майкрософт оставляет за собой право отказать в архивной обработке в тех случаях, когда архивный почтовый ящик пользователя используется для хранения архивных данных для других пользователей или в других случаях ненадлежащего использования.

## <a name="auto-expanding-archiving"></a>Автоматическое расширение архива

Функция архивирования под названием *автоматическое расширение* архивирования предоставляет дополнительное пространство для хранения в архивных почтовых ящиках. Каждый подписчик Exchange Online Archiving изначально получает хранилище объемом 100 ГБ в архивном почтовом ящике. При автоматическом расширении архивирования автоматически добавляется дополнительное пространство для хранения, когда достигается емкость хранилища в 100 ГБ. Это постепенное добавление пространства для хранения продолжается до тех пор, пока хранилище архива не достигнет 1,5 ТБ. В Exchange гибридных развертываниях автоматическое расширение архивации поддерживается только для облачных архивных почтовых ящиков, когда почтовый ящик локального пользователя находится в Exchange Server 2019, Exchange Server 2016 г. или Exchange Server 2013 г. (SP1 или более поздней). Дополнительные сведения см. в статье [Обзор архивации с автоматическим расширением](/microsoft-365/compliance/autoexpanding-archiving).
  
> [!IMPORTANT]
> Администраторы не могут настраивать квоту хранилища.
>
> Автоматическое расширение архива не поддерживается для почтовых ящиков, Exchange Server 2010 г.

> Архив автоматического расширения поддерживается только для почтовых ящиков, используемых для отдельных пользователей или общих почтовых ящиков со скоростью роста не более *1 &nbsp; ГБ в день.* Не разрешается использовать функцию ведения журнала, правила транспорта и правила автоматической пересылки, чтобы копировать сообщения на Архивация на базе Exchange Online для создания архива. Архивный почтовый ящик пользователя предназначен только для этого пользователя. Корпорация Майкрософт оставляет за собой право отказать в дополнительном хранилище архива в тех случаях, когда архивный почтовый ящик пользователя используется для хранения архивных данных для других пользователей или в других случаях ненадлежащего использования.
